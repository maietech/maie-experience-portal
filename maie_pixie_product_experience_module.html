The premise

What the Pixie is not

Not a chatbot avatar
It does not represent a conversational agent. It represents the user's entire MAIE collaboration — their identity, their work, their history.
Not a mascot
The Pixie is a real-time data visualization. Every animation, every echo, every particle response encodes live information from the agent pipeline.
What it actually is
A workflow mirror
Energy, animation speed, and ring behavior change in real time as agent plans move through planning → execution → completion.
A reputation record
Achievement echoes orbit the canvas permanently. The longer a creator works on MAIE, the richer their Pixie's constellation becomes.
A personal identity
Archetype, theme, name, and marketplace skins make every Pixie distinct. No two users see the same companion.

Identity architecture

Three layers. One presence.

The architecture separates permanence from expression. The innermost layer never changes. Everything outward can be customized, upgraded, or replaced — without touching what matters.

Tier 1 · permanent
UserIdentity
Immutable provenance. The glyph seed that stamps every asset the creator publishes. Changing archetype or theme never touches this layer. This is what other creators see when they encounter your work.
identity_id glyph seed provenance stamp
Tier 2 · expressive
Pixie Companion
Personality, appearance, and character. The archetype defines how the Pixie moves and what role it plays. Theme colors and marketplace skins layer on top. Fully replaceable — swapping archetype never affects creator provenance.
archetype theme_id pixie_name preferences
Tier 3 · live
Runtime intelligence
The Pixie's current emotional and operational state. Phase, mode, progress, and temperament are driven directly by the Agent Workflow Panel in real time. Never persisted — this layer lives only while the agent runs.
phase mode progress % temperament

Extending creative identity

The marketplace is the Pixie's wardrobe

Creators can purchase asset packs that alter their companion's appearance, animation, and feel. This is not monetization for its own sake — it is the mechanism by which experienced creators express their platform identity.

Identity themes
Full animation + shape + palette bundles. One equip changes everything.
Premium skins
AGENT_SKIN images composited onto the canvas, clipped to the active core shape.
Animation packs
New orbit, neural, and wave behaviors with custom particle physics.
Achievement collections
Curated echo sets that recognize specific career milestones and contributions.
Asset ownership gate
Every equippable asset is validated server-side before any DB write. Users can only equip items they own. The API returns 403 on unowned attempts — not a client-side check.
Skin compositing
Skin images fade in over ~40 frames, clipped to the user's chosen core shape. Procedural gradient layers render beneath, so the skin reads as texture rather than a flat overlay.
